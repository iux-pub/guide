디자이너가 피그마에서 작업할 때 따라야 할 네이밍, 구조, 토큰 규칙을 정의한다. 이 컨벤션을 준수하면 디자인-퍼블리싱 간 핸드오프가 원활해지고, 토큰 기반의 일관된 디자인을 유지할 수 있다.

## 가이드 목록

| 가이드 | 설명 |
|--------|------|
| [컴포넌트 네이밍](/figma/component-naming/) | 컴포넌트 계층 구조, 구분자, 대소문자 규칙 |
| [레이어/프레임 구조](/figma/layer-structure/) | 페이지 구성, 프레임 네이밍, 레이어 정리 원칙 |
| [Variable 네이밍](/figma/variables/) | Variable 이름 규칙과 CSS Custom Property 매핑 |
| [Auto Layout 규칙](/figma/auto-layout/) | 패딩/갭 토큰 매핑, 반응형 동작 정의 |

## 핵심 원칙

1. **토큰 기반 디자인**: 모든 색상, 간격, 타이포그래피 값은 Variable(토큰)을 사용한다. 하드코딩 값을 사용하지 않는다.
2. **일관된 네이밍**: 컴포넌트, 레이어, Variable 모두 팀이 합의한 네이밍 규칙을 따른다.
3. **퍼블리싱 연결**: 피그마의 구조가 HTML/CSS 코드 구조와 자연스럽게 매핑되도록 설계한다.
4. **정리된 파일**: 프레임과 레이어를 체계적으로 정리하여 다른 팀원이 파일을 쉽게 이해할 수 있게 한다.
