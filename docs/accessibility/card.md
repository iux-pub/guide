# 카드 접근성 가이드

> 스니펫 코드는 `src/snippets/card.md` 참조. 이 문서는 접근성 심화 가이드(왜/언제/주의사항).

## 필수 ARIA 속성

| 속성 | 대상 요소 | 값 | 설명 |
|------|----------|-----|------|
| `alt` | 카드 내 `<img>` | 이미지 설명 텍스트 | 장식용 이미지는 `alt=""` (빈 값). 정보 전달 이미지는 내용을 설명 |
| `aria-label` | 링크 카드(전체 클릭) | 카드 제목 텍스트 | 카드 전체를 `<a>`로 감싸는 경우 링크 목적을 명시 |

## 키보드 상호작용

| 키 | 동작 |
|----|------|
| `Tab` | 카드 내 링크/버튼으로 포커스 이동 |
| `Enter` | 포커스된 링크/버튼 활성화 |
| `Shift+Tab` | 이전 포커스 가능 요소로 이동 |

## 카드 이미지 alt 작성 원칙

**왜 중요한가:** 스크린리더 사용자는 이미지를 볼 수 없다. alt 텍스트가 유일한 정보 전달 수단이다.

| 이미지 유형 | alt 처리 | 예시 |
|------------|---------|------|
| 정보 전달 (제품 사진) | 구체적 설명 | `alt="흰색 러닝화 측면 사진"` |
| 장식용 (배경 패턴) | 빈 alt | `alt=""` |
| 아이콘 (카테고리 표시) | 의미 설명 | `alt="뉴스 카테고리"` |
| 썸네일 (블로그 글) | 글 내용 요약 또는 빈 alt | 제목이 충분하면 `alt=""` |

**빈 alt(`alt=""`)가 아닌 alt 속성 누락은 금지한다:** alt 속성이 아예 없으면 스크린리더가 파일명을 읽는다 ("IMG underscore 20240325 underscore 001 dot jpeg").

## 링크 카드의 클릭 영역 처리

**왜 주의해야 하는가:** 카드 전체를 `<a>` 태그로 감싸면 스크린리더가 카드 내 모든 텍스트를 하나의 긴 링크 텍스트로 읽는다.

**권장 패턴:** 제목에만 링크를 걸고 CSS `::after` pseudo-element로 클릭 영역을 카드 전체로 확장한다.

```css
/* 이 패턴은 _card.scss에 이미 구현되어 있음 */
.card { position: relative; }
.card__link::after {
  content: '';
  position: absolute;
  inset: 0;
}
```

**이 패턴의 접근성 장점:**
- 스크린리더가 제목 텍스트만 링크로 읽어 간결하다
- 카드 내 다른 링크/버튼에 별도로 접근 가능하다
- 키보드 Tab으로 제목 링크에 포커스, Enter로 이동

## heading 계층 주의사항

**왜 중요한가:** 스크린리더 사용자는 heading 레벨로 페이지 구조를 파악한다. 카드 제목이 항상 `<h3>`일 필요는 없다.

- 카드 목록이 `<h2>` 섹션 안에 있으면 카드 제목은 `<h3>`
- 카드 목록이 `<h3>` 섹션 안에 있으면 카드 제목은 `<h4>`
- heading 레벨을 건너뛰지 않는다 (h2 다음 h4 금지)

## Do / Don't

### Do (올바른 예)

```html
<!-- 제목 링크 + ::after 클릭 영역 확장 -->
<article class="card">
  <div class="card__image">
    <img src="thumbnail.jpg" alt="">
  </div>
  <div class="card__body">
    <h3 class="card__title">
      <a href="/post/123" class="card__link">웹 접근성 가이드 업데이트</a>
    </h3>
    <p class="card__desc">KWCAG 2.2 기준으로 업데이트된 체크리스트를 확인하세요.</p>
  </div>
</article>
```

**왜:** 스크린리더가 "웹 접근성 가이드 업데이트, 링크"로 간결하게 읽는다. 썸네일은 제목과 중복 정보이므로 `alt=""`로 처리한다.

### Don't (잘못된 예)

```html
<!-- 카드 전체를 a 태그로 감싸기 -->
<a href="/post/123" class="card">
  <img src="thumbnail.jpg" alt="웹 접근성 가이드 업데이트 썸네일">
  <h3>웹 접근성 가이드 업데이트</h3>
  <p>KWCAG 2.2 기준으로 업데이트된 체크리스트를 확인하세요.</p>
</a>

<!-- alt 속성 누락 -->
<img src="thumbnail.jpg">
```

**왜 잘못인가:**
- 전체 `<a>` 래퍼는 스크린리더가 "웹 접근성 가이드 업데이트 썸네일, 웹 접근성 가이드 업데이트, KWCAG 2.2 기준으로 업데이트된 체크리스트를 확인하세요, 링크"로 읽는다. 너무 길다.
- alt 누락 시 파일명이 읽혀 사용자 경험을 크게 해친다

## 스크린리더 테스트 노트

- **NVDA:** "웹 접근성 가이드 업데이트, 링크" -> Tab -> 다음 카드 제목 링크로 이동
- **VoiceOver (macOS):** "링크, 웹 접근성 가이드 업데이트" / 장식 이미지: 건너뜀 (alt="" 시)
- **카드 목록:** "목록, N개 항목" (ul/li 사용 시) -> 각 카드 article로 이동

## KWCAG 2.2 관련 검사항목

| 항목 번호 | 항목명 | 이 컴포넌트에서의 확인 방법 |
|-----------|--------|--------------------------|
| 5.1.1 | 적절한 대체 텍스트 제공 | 카드 이미지에 적절한 alt 속성이 있는지, 장식 이미지는 alt=""인지 확인 |
| 5.4.4 | 콘텐츠 간의 구분 | 카드 간 시각적 구분(보더, 간격)이 색상에만 의존하지 않는지 확인 |
| 6.4.3 | 적절한 링크 텍스트 | 카드 링크 텍스트가 목적지를 설명하는지 확인. "더보기"만 사용하지 않는다 |
