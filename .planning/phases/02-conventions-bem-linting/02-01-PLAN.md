
    2. 프로젝트 루트에 `.stylelintrc.json` 생성. 아래 설정을 정확히 사용:
       ```json
       {
         "extends": ["stylelint-config-standard-scss"],
         "plugins": ["stylelint-selector-bem-pattern"],
         "defaultSeverity": "warning",
         "rules": {
           "scss/selector-class-pattern": [
             "^[a-z][a-z0-9]*(-[a-z0-9]+)*((__[a-z0-9]+(-[a-z0-9]+)*)(--[a-z0-9]+(-[a-z0-9]+)*)?|(--[a-z0-9]+(-[a-z0-9]+)*))?$",
             {
               "resolveNestedSelectors": true,
               "message": "BEM 패턴 위반: \"%s\" -> .block__element--modifier 형태로 작성하세요"
             }
           ],
           "plugin/selector-bem-pattern": [
             {
               "preset": "bem",
               "implicitComponents": [
                 "src/scss/5-objects/**/*.scss",
                 "src/scss/6-components/**/*.scss"
               ]
             },
             {
               "severity": "warning",
               "message": "이 파일의 선택자가 파일명 기반 BEM 블록과 일치하지 않습니다"
             }
           ],
           "selector-class-pattern": null
         }
       }
       ```

       핵심 설정 이유 (per D-04, D-05):
       - `"defaultSeverity": "warning"` — 빌드는 성공하되 경고 로그 출력 (D-04)
       - `"message"` 에 올바른 패턴 안내 포함 (D-05)
       - `"selector-class-pattern": null` — 코어 규칙 비활성화, `scss/selector-class-pattern`으로 대체 (Stylelint 17에서 resolveNestedSelectors가 코어에서 제거됨)
       - `"resolveNestedSelectors": true` — SCSS &__element, &--modifier 중첩 해석 (BEM-03)

    3. package.json의 scripts에 lint 명령 추가 (기존 build:css, watch:css 유지):
       ```json
       "lint:css": "stylelint \"src/scss/**/*.scss\"",
       "lint:css:fix": "stylelint \"src/scss/**/*.scss\" --fix"
       ```
       이로써 `npm run lint:css`와 `npm run lint:css:fix` 사용 가능 (D-06)

    `docs/bem-guide.md` 파일을 생성한다. 모든 주석과 설명은 한국어로 작성.

    문서 구조:

    **1. BEM 개요** (per D-01)
    - Block__Element--Modifier 패턴 설명
    - Block: 독립적인 의미 단위 (`.card`, `.btn`, `.nav`)
    - Element: Block의 구성 요소, 독립 사용 불가 (`__header`, `__title`, `__icon`)
    - Modifier: Block/Element의 변형 (`--primary`, `--large`, `--disabled`)
    - 네이밍 규칙: 소문자, 하이픈 연결, element는 `__`, modifier는 `--`

    **2. BEM 허용 패턴** (per D-02)
    - `.block-name` (소문자, 하이픈 연결 블록)
    - `.block__element` (더블 언더스코어 element)
    - `.block--modifier` (더블 하이픈 modifier)
    - `.block__element--modifier` (element + modifier)
    - 실제 프로젝트 예시: `.grid__col-1`, `.container`, `.sr-only`

    **3. DO / DON'T 예제** (per D-03, BEM-02)
    기존 프로젝트의 실제 잘못된 패턴을 사용:

    | DON'T (기존 패턴) | DO (올바른 BEM) | 이유 |
    |---|---|---|
    | `.btn-primary` | `.btn--primary` | modifier에는 `--` 사용 |
    | `.btn-secondary` | `.btn--secondary` | 동일 |
    | `.btn-outline` | `.btn--outline` | 동일 |
    | `.card-header` | `.card__header` | element에는 `__` 사용 |
    | `.card-header h4` | `.card__title` | 요소 선택자 대신 BEM element 사용 |
    | `.form-group label` | `.form__label` | 요소 선택자 의존 제거 |
    | `.input-box.error` | `.form__input--error` | 별도 클래스 조합 대신 modifier |
    | `.btn-gray` | `.btn--secondary` | 시각적 속성 대신 의미적 이름 |
    | `.card__header__title` | `.card__title` | element 2단계 중첩 금지 |

    **4. SCSS 중첩 BEM 작성 규칙** (BEM-03)
    - `&__element`, `&--modifier` 사용 필수
    - element 2단계 중첩 금지 규칙과 해결 방법
    - 정식 예제 (Research Example 3의 버튼 컴포넌트 SCSS 코드 포함):

    ```scss
    .card {
      display: flex;

      &__header {
        padding: var(--spacing-md);
      }

      &__body {
        flex: 1;
      }

      &--featured {
        border-color: var(--color-primary);
      }

      &__title {
        font-size: var(--font-size-lg);

        &--highlight {
          color: var(--color-primary);
        }
      }
    }
    ```

    **5. ITCSS 레이어별 BEM 적용 범위**
    - 1-settings ~ 4-elements: BEM 적용 대상 아님
    - 5-objects, 6-components: BEM 필수 적용
    - 7-utilities: `.sr-only`, `.hidden` 등 BEM block으로 유효

    **6. Stylelint 자동 검증**
    - `npm run lint:css`로 BEM 위반 자동 검사
    - `npm run lint:css:fix`로 자동 수정 가능한 항목 처리
    - 위반 시 경고 메시지 안내 (D-04, D-05)
    - `.stylelintrc.json` 설정 요약
