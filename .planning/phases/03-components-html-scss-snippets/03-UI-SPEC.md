# Phase 3 — UI Design Contract

> Visual and interaction contract for 8 core UI components + HTML boilerplate. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom ITCSS + BEM) |
| Preset | not applicable |
| Component library | none (vanilla HTML+SCSS+JS) |
| Icon library | none (컴포넌트에 아이콘 미포함, 프로젝트별 선택) |

## Spacing Scale

Declared values (4px 기반 스케일, 62.5% REM 트릭 적용: 1rem = 10px):

| Token | Value | CSS Custom Property | Usage |
|-------|-------|---------------------|-------|
| xs | 4px (0.4rem) | `var(--spacing-xs)` | 아이콘 간격, 인라인 패딩, 버튼 내부 gap |
| sm | 8px (0.8rem) | `var(--spacing-sm)` | 버튼 패딩(세로), 폼 요소 내부 간격 |
| md | 16px (1.6rem) | `var(--spacing-md)` | 기본 요소 간격, 카드 내부 패딩, 테이블 셀 패딩 |
| lg | 24px (2.4rem) | `var(--spacing-lg)` | 섹션 패딩, 모달 내부 여백, playground 미리보기 영역 |
| xl | 32px (3.2rem) | `var(--spacing-xl)` | 레이아웃 간격, 컴포넌트 섹션 간 거리 |
| 2xl | 48px (4.8rem) | `var(--spacing-2xl)` | 주요 섹션 구분, playground 섹션 간 여백 |
| 3xl | 64px (6.4rem) | `var(--spacing-3xl)` | 페이지 레벨 간격 |

Exceptions:
- 최소 터치 영역: 44px x 44px (WCAG 2.1 AA Target Size). 버튼 `--sm` variant의 시각적 높이는 32px이나, 클릭 영역은 `min-height: 4.4rem`으로 보장
- 포커스 아웃라인: `outline-offset: 2px` (스케일 외 고정값)

## Typography

62.5% REM 트릭 적용 (1rem = 10px). 모든 값은 토큰 CSS Custom Property 사용 필수.

**4 sizes, 2 weights** — 이 Phase에서 사용하는 타이포그래피 역할:

| Role | Size | Token | Weight | Token | Line Height | Token |
|------|------|-------|--------|-------|-------------|-------|
| Caption/Label | 14px (1.4rem) | `var(--font-size-sm)` | 400 | `var(--font-weight-regular)` | 1.6 | `var(--leading-base)` |
| Body | 16px (1.6rem) | `var(--font-size-base)` | 400 | `var(--font-weight-regular)` | 1.6 | `var(--leading-base)` |
| Subheading | 20px (2.0rem) | `var(--font-size-md)` | 700 | `var(--font-weight-bold)` | 1.2 | `var(--leading-tight)` |
| Heading/Display | 24px (2.4rem) | `var(--font-size-lg)` | 700 | `var(--font-weight-bold)` | 1.2 | `var(--leading-tight)` |

**통합 안내:** 프로젝트 토큰에는 `--font-size-xs` (12px), `--font-size-xl` (28px), `--font-size-2xl` (32px)과 `--font-weight-medium` (500), `--font-weight-semibold` (600)이 정의되어 있으나, 이 Phase의 8개 컴포넌트는 위 4가지 크기와 2가지 굵기만 사용한다. 프로젝트별 확장 시 추가 토큰 활용 가능.

## Color

모든 색상값은 CSS Custom Property 토큰 사용 필수. 하드코딩 금지.

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `var(--color-bg)` | #fff | 페이지 배경, 모달 배경, 카드 배경, 폼 입력 배경 |
| Secondary (30%) | `var(--color-bg-secondary)` / `var(--color-gray-100)` | #f8f8f8 / #efefef | 테이블 스트라이프 행, 버튼 secondary 배경, 코드 블록 배경, 호버 상태 |
| Accent (10%) | `var(--color-primary)` | #256ef4 | 아래 "Accent reserved for" 목록 참조 |
| Destructive | `var(--color-danger)` | #de3412 | 폼 에러 상태 테두리/메시지, 삭제 확인 버튼 (사용 시) |

### Accent reserved for (10% 제한 -- 아래 요소에만 사용)

- `btn--primary` 배경색
- 폼 입력 포커스 테두리 (`border-color`)
- 탭 활성 상태 하단 인디케이터
- 페이지네이션 현재 페이지 배경
- 카드 `--featured` modifier 테두리
- 포커스 아웃라인 (`outline: 2px solid var(--color-primary)`)
- 링크 기본 색상

### Semantic Colors (접근성 필수 상태)

| State | Token | Value | Usage |
|-------|-------|-------|-------|
| Error | `var(--color-danger)` | #de3412 | 폼 에러 테두리, 에러 메시지 텍스트 |
| Success | `var(--color-success)` | #228738 | 폼 성공 상태 테두리, 성공 메시지 텍스트 |
| Warning | `var(--color-warning)` | #c78500 | 경고 메시지 (이 Phase에서는 미사용) |
| Info | `var(--color-info)` | #0b78cb | 정보 메시지 (이 Phase에서는 미사용) |

## Copywriting Contract

이 Phase는 컴포넌트 라이브러리(스니펫)이므로 "앱 카피"가 아닌 "데모/플레이스홀더 카피"를 정의한다. 실제 프로젝트에서 오버라이드한다.

| Element | Copy | 비고 |
|---------|------|------|
| Primary CTA (btn--primary 데모) | "저장하기" | 한국어 기본, 프로젝트별 변경 |
| Secondary CTA (btn--secondary 데모) | "작성 취소" | |
| Destructive CTA (btn--danger 데모) | "항목 삭제" | |
| 폼 제출 버튼 | "등록하기" | |
| 폼 에러 메시지 | "필수 입력 항목입니다." | `aria-describedby`로 입력과 연결 |
| 폼 성공 메시지 | "입력이 확인되었습니다." | |
| 폼 placeholder | "내용을 입력하세요" | |
| 모달 닫기 버튼 | "닫기" | `aria-label="닫기"` |
| 모달 확인 CTA | "저장하기" | |
| 모달 취소 CTA | "작성 취소" | |
| 테이블 caption (데모) | "사용자 목록" | `<caption>` 필수 제공 |
| 테이블 빈 상태 | "등록된 데이터가 없습니다. 새 항목을 추가해 주세요." | `<td colspan>` 내 표시 |
| 페이지네이션 이전 | "이전" | `aria-label="이전 페이지"` |
| 페이지네이션 다음 | "다음" | `aria-label="다음 페이지"` |
| 페이지네이션 현재 | "현재 페이지" | `aria-current="page"` + `aria-label="현재 페이지, {N}페이지"` |
| 브레드크럼 nav | "현재 위치" | `aria-label="현재 위치"` |
| 브레드크럼 현재 항목 | (페이지명) | `aria-current="page"` |
| Skip to content | "본문 바로가기" | 보일러플레이트 필수 요소 |
| 탭 (데모 레이블) | "탭 {N}" | `role="tab"`, `aria-selected` |
| Playground 페이지 제목 | "{컴포넌트명} 컴포넌트 - 미리보기" | `<title>` 태그 |

### Destructive Action Confirmation

| Action | Confirmation Approach |
|--------|----------------------|
| 모달 닫기 (일반) | 확인 없이 ESC/닫기 버튼으로 즉시 닫기 |
| 삭제 확인 모달 (데모) | 별도 확인 모달: "정말 삭제하시겠습니까?" + "항목 삭제" (danger) / "작성 취소" 버튼 |

|-------|-----------------|-------|

## Playground Page Layout

각 컴포넌트별 `src/playground/*.html` 미리보기 페이지의 레이아웃 계약.

주요 시각적 앵커: `.pg__title` (h1, 가장 큰 타이포그래피 요소 -- `--font-size-lg` 24px bold). 페이지 최상단에 위치하며, 사용자가 어떤 컴포넌트 페이지에 있는지 즉시 인지하는 초점 역할을 한다.

### 구조

```
.pg                          -- 최대 너비 1200px, 중앙 정렬
  .pg__title                 -- 컴포넌트명 (h1, 24px bold)
  .pg__section               -- variant별 섹션 (margin-bottom: 48px)
    .pg__subtitle            -- variant명 (h2, 20px bold)
    .pg__preview             -- 렌더링 미리보기 영역 (padding: 24px, 연한 테두리)
    .pg__code                -- HTML 소스코드 (<pre><code>, 배경 #f8f8f8)
```

### Playground 전용 스타일 규칙

- `pg__` 접두사 클래스만 인라인 `<style>` 태그 허용 (BEM 예외: playground는 가이드 문서 전용)
- 컴포넌트 자체 스타일은 `dist/css/style.css`에서 로드
- 하드코딩 값 허용 (playground 레이아웃은 토큰 시스템 외부)
