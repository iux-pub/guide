# 브레드크럼 접근성 가이드

> 스니펫 코드는 `src/snippets/breadcrumb.md` 참조. 이 문서는 접근성 심화 가이드(왜/언제/주의사항).

## 필수 ARIA 속성

| 속성 | 대상 요소 | 값 | 설명 |
|------|----------|-----|------|
| `aria-label="현재 위치"` | `<nav>` | 텍스트 | 브레드크럼 네비게이션의 목적을 명시 |
| `aria-current="page"` | 마지막 항목 (현재 페이지) | `page` | 현재 보고 있는 페이지임을 스크린리더에 전달 |

## 키보드 상호작용

| 키 | 동작 |
|----|------|
| `Tab` | 브레드크럼 내 다음 링크로 포커스 이동 |
| `Shift+Tab` | 이전 링크로 포커스 이동 |
| `Enter` | 해당 페이지로 이동 |

## 구분자 처리

**왜 중요한가:** 브레드크럼 항목 사이의 구분자(>, /, >)를 스크린리더가 읽으면 사용자 경험이 나빠진다. "홈, 보다 크다, 게시판, 보다 크다, 공지사항"처럼 읽히기 때문이다.

**올바른 처리 방법:**

1. **CSS `::before` pseudo-element (권장):** DOM에 구분자가 없으므로 스크린리더가 자동으로 무시한다. 이 프로젝트의 `_breadcrumb.scss`가 이 방식을 사용한다.

2. **`aria-hidden="true"` (차선):** HTML에 구분자를 넣어야 하는 경우 `<span aria-hidden="true">/</span>`으로 숨긴다.

3. **`<ol>` 순서 목록 활용:** `<ol>` 안의 `<li>`는 스크린리더가 "N개 항목 중 M번째"로 읽어 구분자 없이도 계층 구조를 전달한다.

## 현재 페이지 항목: 링크 vs 텍스트

**왜 구분하는가:** 브레드크럼의 마지막 항목은 현재 페이지이므로, 이미 그 페이지에 있다. 클릭해서 이동할 곳이 없다.

| 방식 | 구현 | 설명 |
|------|------|------|
| `<span>` (권장) | `<span aria-current="page">공지사항</span>` | 링크가 아니므로 혼란 없음 |
| `<a>` + `aria-current` | `<a href="#" aria-current="page">공지사항</a>` | 링크지만 현재 페이지임을 명시 |

## 모바일에서의 접근성 고려

**상위 경로 숨김:** 모바일에서 `.breadcrumb__item--mobile-hidden`으로 중간 경로를 숨길 때, `display: none`이나 `visibility: hidden`을 사용하면 스크린리더도 읽지 못한다.

**접근성 관점에서의 선택:**
- 화면 공간 절약이 목적이면 **시각적 숨김은 허용**하되, 스크린리더는 전체 경로를 읽을 수 있도록 `.sr-only` 패턴을 고려한다
- 현재 구현은 `display: none`으로 스크린리더에서도 숨겨진다. 이는 모바일에서도 "현재 위치 > 현재 페이지" 수준의 정보만 제공하겠다는 의도적 선택이다

## Do / Don't

### Do (올바른 예)

```html
<nav class="breadcrumb" aria-label="현재 위치">
  <ol class="breadcrumb__list">
    <li class="breadcrumb__item">
      <a href="/" class="breadcrumb__link">홈</a>
    </li>
    <li class="breadcrumb__item">
      <a href="/board" class="breadcrumb__link">게시판</a>
    </li>
    <li class="breadcrumb__item">
      <span class="breadcrumb__current" aria-current="page">공지사항</span>
    </li>
  </ol>
</nav>
```

**왜:** `<nav>` + `aria-label`로 목적 전달, `<ol>`로 계층 구조, CSS `::before`로 구분자 (DOM에 없음), 현재 페이지는 `<span>` + `aria-current="page"`.

### Don't (잘못된 예)

```html
<!-- nav 없음, 구분자가 DOM에 직접 삽입 -->
<div class="breadcrumb">
  <a href="/">홈</a>
  <span>></span>
  <a href="/board">게시판</a>
  <span>></span>
  <span>공지사항</span>
</div>
```

**왜 잘못인가:**
- `<nav>` 없이는 스크린리더가 네비게이션 영역을 인식하지 못한다
- `<ol>` / `<li>` 없이는 계층 구조를 전달할 수 없다
- `<span>></span>` 구분자를 스크린리더가 "보다 크다"로 읽는다
- 현재 페이지에 `aria-current="page"` 없으면 어떤 항목이 현재 페이지인지 알 수 없다

## 스크린리더 테스트 노트

- **NVDA:** "현재 위치, 네비게이션" -> "목록, 3개 항목" -> "홈, 링크, 3개 중 1번째" -> "게시판, 링크, 3개 중 2번째" -> "공지사항, 현재 페이지, 3개 중 3번째"
- **VoiceOver (macOS):** "현재 위치, 탐색" -> "목록 3개 항목" -> "홈, 링크" -> "게시판, 링크" -> "공지사항, 현재 페이지"
- **구분자:** CSS `::before`로 구현된 구분자는 스크린리더가 읽지 않음 확인
