각 파일 상단에 `@use '../2-tools/responsive' as resp;`를 추가한다 (이미 있는 form은 생략). 모바일 퍼스트로 기본값은 모바일 패딩이고, tablet/pc에서 확대한다.

**btn.scss** (per RESP-01):
- 기존 패딩: `padding: var(--spacing-sm) var(--spacing-md)` (8px 16px) — 이것을 모바일 기본값으로 유지
- tablet에서: `padding: var(--spacing-sm) var(--spacing-lg)` (8px 24px)
- pc에서: `padding: var(--spacing-md) var(--spacing-lg)` (16px 24px)
- `&--sm`의 패딩도: 모바일 기본 유지, tablet에서 `padding: var(--spacing-xs) var(--spacing-md)` (4px 16px)
- `&--lg`의 패딩도: 모바일 기본 유지, tablet에서 `padding: var(--spacing-md) var(--spacing-xl)` (16px 32px), pc에서 `padding: var(--spacing-lg) var(--spacing-xl)` (24px 32px)
- respond-to 블록은 각 셀렉터 내부에 중첩한다

**form.scss** (per RESP-01, SIZE-01):
- `&__label`의 `font-size`를 `var(--font-size-sm)` -> `var(--font-size-base)` (14px -> 16px)로 변경 (SIZE-01)
- `&__input, &__select, &__textarea` 패딩: 기존 `var(--spacing-sm) var(--spacing-md)` (8px 16px) 모바일 유지
  - tablet에서: `padding: var(--spacing-sm) var(--spacing-lg)` (8px 24px)
  - pc에서: `padding: var(--spacing-md) var(--spacing-lg)` (16px 24px)
- `&__group` margin-bottom: 모바일 기본 유지, pc에서 `margin-bottom: var(--spacing-lg)` (24px)

**table.scss** (per RESP-01, SIZE-01):
- `@use '../2-tools/responsive' as resp;`를 파일 상단에 추가
- `.table`의 `font-size`를 `var(--font-size-sm)` -> `var(--font-size-base)` (14px -> 16px)로 변경 (SIZE-01)
- `&__th, &__td` 패딩: 기존 `var(--spacing-md)` (16px) 모바일 유지
  - tablet에서: `padding: var(--spacing-md) var(--spacing-lg)` (16px 24px)
  - pc에서: `padding: var(--spacing-lg)` (24px)
- `&__empty` 패딩: 모바일 기본 유지, pc에서 `padding: var(--spacing-3xl)` (64px)

respond-to 블록 작성 패턴:
```scss
&__th {
  padding: var(--spacing-md);

  @include resp.respond-to('tablet') {
    padding: var(--spacing-md) var(--spacing-lg);
  }

  @include resp.respond-to('pc') {
    padding: var(--spacing-lg);
  }
}
```

주의: 기존의 prefers-reduced-motion, :focus-visible, BEM 구조를 절대 변경하지 않는다. respond-to 블록만 추가한다.
