모든 컴포넌트는 3단계 라이프사이클을 거친다. 각 단계는 컴포넌트의 성숙도와 사용 범위를 나타내며, 프로덕션 적용 가능 여부를 판단하는 기준이 된다.

## 라이프사이클 3단계

### experimental (실험)

새로운 아이디어나 패턴을 탐색하는 초기 단계이다.

| 항목 | 내용 |
|------|------|
| **정의** | 유즈케이스가 제안되었으나 팀 검증을 거치지 않은 컴포넌트 |
| **진입 조건** | 유즈케이스 1개 이상 제안, GitHub 이슈 등록 |
| **사용 제한** | 프로덕션 사용 금지. 프로토타입이나 내부 테스트에서만 사용 가능 |
| **라벨 표기** | 문서: `[experimental]` 태그, 코드: 주석에 `// [experimental]` 표기 |

### beta (베타)

팀 리뷰를 통과하고 제한적으로 프로덕션에 적용 가능한 단계이다.

| 항목 | 내용 |
|------|------|
| **정의** | 팀 리뷰를 통과하고 기본 품질 기준을 충족한 컴포넌트 |
| **진입 조건** | 팀 리뷰 통과, 기본 접근성 검증(KWCAG/WCAG AA), HTML+SCSS 스니펫 작성 완료 |
| **사용 제한** | 프로덕션 사용 가능하나 API 변경 가능성 있음. 사용 시 변경 추적 필요 |
| **라벨 표기** | 문서: `[beta]` 태그, 코드: 주석에 `// [beta]` 표기 |

### stable (안정)

충분한 검증을 거쳐 팀 표준으로 확정된 단계이다.

| 항목 | 내용 |
|------|------|
| **정의** | 다수 프로젝트에서 검증되고 팀 표준으로 확정된 컴포넌트 |
| **진입 조건** | 3개 이상 프로젝트 적용, pa11y 접근성 검사 통과, 문서(스니펫 + 접근성 가이드) 완비 |
| **사용 제한** | 제한 없음. 모든 프로젝트에서 자유롭게 사용 가능 |
| **라벨 표기** | 문서: 별도 태그 없음(기본 상태), 코드: 표기 불필요 |

## 승격 조건

### experimental -> beta

다음 조건을 **모두** 충족해야 한다:

1. 팀 리뷰 통과 (2인 이상 동의)
2. BEM 네이밍 컨벤션 준수
3. 기본 접근성 검증 (KWCAG/WCAG AA)
4. HTML 마크업 + SCSS 스타일 스니펫 작성 완료
5. 반응형 동작 확인 (모바일/태블릿/PC)

### beta -> stable

다음 조건을 **모두** 충족해야 한다:

1. 3개 이상 실제 프로젝트에서 적용 및 검증
2. pa11y 접근성 검사 통과
3. 문서 완비 (스니펫, 접근성 가이드, 사용법)
4. 크로스 브라우저 테스트 통과
5. 팀 전체 합의

## 강등 조건

### stable -> deprecated (지원 중단)

다음 조건 중 **하나** 이상 해당 시:

- 대체 컴포넌트가 stable로 승격됨
- 더 이상 사용되지 않는 패턴
- 접근성 기준을 구조적으로 충족할 수 없음

**deprecated 처리 규칙:**

1. CHANGELOG에 Deprecated로 기록
2. 문서에 `[deprecated]` 태그와 대체 컴포넌트 안내 추가
3. 코드에 `// [deprecated] -> 대체 컴포넌트명 사용` 주석 추가
4. **6개월 유예 기간** 제공 (기존 프로젝트 마이그레이션 기간)

### deprecated -> 삭제

- **MAJOR 버전 업데이트에서만** 삭제 가능
- 삭제 전 마이그레이션 가이드 제공 필수
- CHANGELOG에 Removed로 기록

## 현재 컴포넌트 상태

Phase 3~4에서 HTML+SCSS 스니펫 작성 및 접근성 검증을 완료한 8개 컴포넌트 현황이다.

| 컴포넌트 | 상태 | 비고 |
|----------|------|------|
| btn (버튼) | stable | Phase 3에서 구현, Phase 4에서 접근성 검증 완료 |
| form (폼) | stable | Phase 3에서 구현, Phase 4에서 접근성 검증 완료 |
| card (카드) | stable | Phase 3에서 구현, Phase 4에서 접근성 검증 완료 |
| table (테이블) | stable | Phase 3에서 구현, Phase 4에서 접근성 검증 완료 |
| modal (모달) | stable | Phase 3에서 구현, Phase 4에서 접근성 검증 완료 |
| tab (탭) | stable | Phase 3에서 구현, Phase 4에서 접근성 검증 완료 |
| pagination (페이지네이션) | stable | Phase 3에서 구현, Phase 4에서 접근성 검증 완료 |
| breadcrumb (브레드크럼) | stable | Phase 3에서 구현, Phase 4에서 접근성 검증 완료 |
