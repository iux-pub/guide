# KWCAG/WCAG 2.1 AA 퍼블리싱 체크리스트

> KWCAG 2.2 확대 적용 (33개 검사항목 기준, 2026년 전자정부 웹사이트 의무)
> Phase 3 컴포넌트 구조와 1:1 대응하는 컴포넌트별 분류

## 사용법

1. 새 프로젝트 시작 시 이 파일을 복사하여 프로젝트 루트에 배치한다
2. 컴포넌트 퍼블리싱 완료 후 해당 섹션의 체크박스를 확인한다
3. `[NEW]` 표시 항목은 KWCAG 2.2에서 신규 추가된 검사항목이다
4. "해당 없음" 섹션의 항목은 현재 컴포넌트에 해당하지 않으나, 프로젝트 확장 시 재확인이 필요하다
5. 자동 검사: `npm run test:a11y` (pa11y-ci, WCAG2AA 기준)
6. 개별 페이지 검사: `npx axe <URL>`

---

## 공통 항목

모든 페이지와 컴포넌트에 공통 적용되는 검사항목이다.

- [ ] **5.1.1 적절한 대체 텍스트 제공** -- 이미지에 `alt` 속성 필수, 장식용 이미지는 `alt=""` + `role="presentation"`, 의미 있는 이미지는 내용을 설명하는 텍스트 제공
- [ ] **5.3.2 콘텐츠의 선형구조** -- CSS를 제거해도 콘텐츠 순서가 논리적, 시각적 배치와 DOM 순서 일치, `tabindex` 양수값 사용 금지
- [ ] **5.4.1 색에 무관한 콘텐츠 인식** -- 색상만으로 정보를 전달하지 않음, 오류/성공 상태에 아이콘 또는 텍스트 병행 표시
- [ ] **5.4.3 텍스트 콘텐츠의 명도 대비** -- 일반 텍스트 4.5:1 이상, 큰 텍스트(24px/18px bold) 3:1 이상, 토큰 색상 조합 기준 `docs/accessibility/color-contrast.md` 참조
- [ ] **6.1.2 초점 이동과 표시** -- 모든 인터랙티브 요소에 `focus-visible` 스타일 제공, 초점 순서가 논리적, 초점이 화면 밖으로 벗어나지 않음
- [ ] **6.4.2 제목 제공** -- 페이지 `<title>` 제공, 제목 레벨 순서 준수 (`h1` > `h2` > `h3`, 건너뛰기 금지), 콘텐츠 영역별 적절한 제목 레벨 사용
- [ ] **7.1.1 기본 언어 표시** -- `<html lang="ko">` 속성 필수 지정, 외국어 인용 시 해당 요소에 `lang` 속성 추가
- [ ] **8.1.1 마크업 오류 방지** -- 열고 닫는 태그 쌍 일치, `id` 속성 중복 없음, 속성 값 따옴표 처리, W3C Validator 통과

## 버튼 (btn)

해당 검사항목: 5.4.1, 6.1.1, 6.1.3, 6.5.3

- [ ] **5.4.1 색에 무관한 콘텐츠 인식** -- 버튼 상태(활성/비활성)를 색상만으로 구분하지 않음, `disabled` 속성 + `opacity` 변경 병행
- [ ] **6.1.1 키보드 사용 보장** -- 모든 버튼이 `Tab` 키로 접근 가능, `Enter`/`Space` 키로 동작, 네이티브 `<button>` 태그 사용 필수
- [ ] **6.1.3 조작 가능** -- 최소 터치 영역 44px x 44px 보장 (`.btn--sm`도 `min-height: 4.4rem`), 버튼 간 간격 충분
- [ ] **6.5.3 레이블과 네임** `[NEW]` -- 아이콘만 있는 버튼에 `aria-label` 필수, 버튼 텍스트가 시각적 레이블과 일치, `aria-label`은 시각적 텍스트를 포함

---

## 폼 (form)

해당 검사항목: 5.3.3, 5.4.1, 6.1.1, 6.5.3, 7.2.1, 7.3.1, 7.3.2

- [ ] **5.3.3 명확한 지시사항 제공** -- 입력 형식 안내 텍스트 제공 (예: "YYYY-MM-DD"), 필수 항목 표시에 색상 외 `*` 또는 "(필수)" 텍스트 병행
- [ ] **5.4.1 색에 무관한 콘텐츠 인식** -- 유효성 검사 오류를 색상만으로 표시하지 않음, 아이콘 + 텍스트 메시지 병행 (`.form__message--error`)
- [ ] **6.1.1 키보드 사용 보장** -- 모든 입력 필드, 셀렉트, 체크박스, 라디오에 `Tab` 키 접근 가능, 폼 제출이 `Enter` 키로 동작
- [ ] **6.5.3 레이블과 네임** `[NEW]` -- `<label>`의 `for` 속성이 입력 필드 `id`와 연결, `aria-label` 사용 시 시각적 레이블 텍스트 포함
- [ ] **7.2.1 사용자 요구에 따른 실행** -- 포커스 이동만으로 자동 제출되지 않음, 값 변경만으로 페이지 전환 없음, 사용자 명시적 동작(버튼 클릭) 후 실행
- [ ] **7.3.1 오류 정정** -- 오류 발생 시 오류 항목 식별 + 수정 방법 안내, 오류 메시지가 해당 필드 근처에 표시, `aria-describedby`로 오류 메시지 연결
- [ ] **7.3.2 레이블 제공** -- 모든 입력 필드에 `<label>` 연결 필수, 그룹 입력은 `<fieldset>` + `<legend>` 사용, placeholder만으로 레이블 대체 금지

## 카드 (card)

해당 검사항목: 5.1.1, 5.4.4, 6.4.3

- [ ] **5.1.1 적절한 대체 텍스트 제공** -- 카드 내 이미지에 `alt` 텍스트 제공, 장식용 썸네일은 `alt=""` + `role="presentation"`
- [ ] **5.4.4 콘텐츠 간의 구분** -- 카드 간 시각적 경계 명확 (테두리 또는 배경색 차이), 색상만으로 구분하지 않고 border 등 병행
- [ ] **6.4.3 적절한 링크 텍스트** -- 카드 내 링크 텍스트가 목적지를 설명 ("더보기" 대신 "공지사항 더보기"), `aria-label`로 구체적 링크 목적 제공 가능

---

## 테이블 (table)

해당 검사항목: 5.3.1, 5.4.4, 6.1.1

- [ ] **5.3.1 표의 구성** -- `<caption>` 제공 필수 (시각적 숨김 가능: `.sr-only`), `<thead>` + `<th scope="col">` 열 제목 명시, 행 제목 필요 시 `<th scope="row">` 사용
- [ ] **5.4.4 콘텐츠 간의 구분** -- 행 간 구분선 또는 교대 배경색 제공, 셀 간 경계 명확, 데이터 없는 셀에 "-" 또는 "해당 없음" 표시
- [ ] **6.1.1 키보드 사용 보장** -- 테이블 내 링크/버튼이 `Tab` 키로 접근 가능, 정렬 버튼 등 인터랙티브 요소 키보드 동작 보장

---

## 모달 (modal)

해당 검사항목: 5.3.2, 6.1.1, 6.1.3, 7.2.1, 8.2.1

- [ ] **5.3.2 콘텐츠의 선형구조** -- 모달 열림 시 포커스가 모달 내부로 이동, 모달 닫힘 시 포커스가 트리거 요소로 복귀
- [ ] **6.1.1 키보드 사용 보장** -- `Esc` 키로 모달 닫기, 모달 내부에서 `Tab` 키 순환 (포커스 트랩), 닫기 버튼에 키보드 접근 가능
- [ ] **6.1.3 조작 가능** -- 닫기 버튼 최소 44px x 44px, 백드롭 클릭으로 닫기 가능, 닫기 버튼에 `aria-label="닫기"` 제공
- [ ] **7.2.1 사용자 요구에 따른 실행** -- 모달이 자동으로 열리지 않음 (사용자 동작에 의해서만 열림), 닫기 동작이 예측 가능
- [ ] **8.2.1 웹 애플리케이션 접근성 준수** -- `role="dialog"` + `aria-modal="true"` 필수, `aria-labelledby`로 모달 제목 연결, `aria-describedby`로 설명 연결 (선택)

---

## 탭 (tab)

해당 검사항목: 6.1.1, 6.1.3, 6.4.2, 8.2.1

- [ ] **6.1.1 키보드 사용 보장** -- 좌우 화살표 키로 탭 전환, `Tab` 키로 탭 패널 내부 콘텐츠 접근, `Home`/`End` 키로 첫번째/마지막 탭 이동
- [ ] **6.1.3 조작 가능** -- 탭 버튼 최소 터치 영역 보장, 탭 전환 시 화면 깜빡임 없음
- [ ] **6.4.2 제목 제공** -- 탭 그룹에 적절한 레이블 제공 (`aria-label` 또는 연결된 제목)
- [ ] **8.2.1 웹 애플리케이션 접근성 준수** -- `role="tablist"` + `role="tab"` + `role="tabpanel"` 구조 필수, `aria-selected="true/false"` 활성 상태 표시, `aria-controls`로 탭-패널 연결, `tabindex` 로빙 관리 (활성 탭: `0`, 비활성: `-1`)

---

## 페이지네이션 (pagination)

해당 검사항목: 6.1.1, 6.4.3, 5.1.1

- [ ] **6.1.1 키보드 사용 보장** -- 모든 페이지 링크에 `Tab` 키 접근 가능, 현재 페이지 표시 요소에 키보드 접근 가능
- [ ] **6.4.3 적절한 링크 텍스트** -- 숫자만 있는 링크에 `aria-label="N 페이지"` 제공, 현재 페이지에 `aria-current="page"` 표시, "이전"/"다음" 버튼 텍스트 또는 `aria-label` 제공
- [ ] **5.1.1 적절한 대체 텍스트 제공** -- 화살표 아이콘에 `aria-label` 또는 `.sr-only` 텍스트 제공, `<nav aria-label="페이지 탐색">` 랜드마크 제공

---

## 브레드크럼 (breadcrumb)

해당 검사항목: 6.4.3, 5.3.2, 6.1.1

- [ ] **6.4.3 적절한 링크 텍스트** -- 각 링크가 해당 페이지를 명확히 설명, 현재 페이지는 링크가 아닌 텍스트로 표시 (`aria-current="page"`)
- [ ] **5.3.2 콘텐츠의 선형구조** -- 브레드크럼 순서가 사이트 계층과 일치, `<nav aria-label="현재 위치">` 랜드마크 제공, `<ol>` 순서 목록 사용 권장
- [ ] **6.1.1 키보드 사용 보장** -- 모든 브레드크럼 링크에 `Tab` 키 접근 가능, 구분자(/)는 장식용이므로 `aria-hidden="true"` 처리

---

## 보일러플레이트 (boilerplate)

해당 검사항목: 6.4.1, 7.1.1

- [ ] **6.4.1 반복 영역 건너뛰기** -- 본문 건너뛰기 링크 제공: `<a href="#main-content" class="skip-to-content">본문 바로가기</a>`, 페이지 최상단에 배치, 포커스 시 화면에 표시
- [ ] **7.1.1 기본 언어 표시** -- `<html lang="ko">` 필수 지정, 다국어 콘텐츠 포함 시 해당 영역에 `lang` 속성 추가

## 해당 없음 항목 (참고)

현재 컴포넌트에 해당하지 않는 검사항목이다. 프로젝트 확장 시 해당 기능이 추가되면 재검토가 필요하다.

| 번호 | 검사항목 | 비해당 사유 |
|------|---------|------------|
| 5.2.1 | 자막 제공 | 멀티미디어 콘텐츠 없음 |
| 5.4.2 | 자동 재생 금지 | 자동 재생 미디어 없음 |
| 6.1.4 | 문자 단축키 `[NEW]` | 문자 단축키 미사용 |
| 6.2.1 | 응답시간 조절 | 시간 제한 기능 없음 |
| 6.2.2 | 정지 기능 제공 | 자동 갱신/이동 콘텐츠 없음 |
| 6.3.1 | 깜빡임과 번쩍임 사용 제한 | 깜빡임/번쩍임 효과 없음 |
| 6.4.4 | 고정된 참조 위치 정보 `[NEW]` | 단일 페이지 구조 (복수 페이지 탐색 없음) |
| 6.5.1 | 단일 포인터 입력 지원 `[NEW]` | 드래그 등 복합 포인터 조작 없음 |
| 6.5.2 | 포인터 입력 취소 `[NEW]` | 포인터 기반 커스텀 조작 없음 |
| 6.5.4 | 동작기반 작동 `[NEW]` | 기기 동작(흔들기 등) 기반 기능 없음 |
| 7.2.2 | 찾기 쉬운 도움 정보 `[NEW]` | 가이드 사이트 자체가 도움 정보 |
| 7.3.3 | 접근 가능한 인증 `[NEW]` | 인증 기능 없음 |
| 7.3.4 | 반복 입력 정보 `[NEW]` | 반복 입력 요구 기능 없음 |
