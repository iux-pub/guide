인포마인드 UX팀의 디자인 원칙과 실무 기준을 정리한 가이드. 피그마 디자인 단계에서 퍼블리싱까지 일관된 품질을 유지하기 위한 디자인 의사결정 기준을 제공한다.

## 가이드 목록

| 가이드 | 설명 |
|--------|------|
| [마이크로카피](/design/microcopy/) | 버튼, 오류 메시지, 플레이스홀더, 빈 상태 등 UI 텍스트 작성 기준 |
| [인터랙션 타이밍](/design/interaction-timing/) | 전환 시간, 이징 함수, 컴포넌트별 모션 명세와 접근성 대응 |
| [디자인 감사](/design/design-audit/) | 17점 체크리스트와 Quick 5 납품 전 점검 기준 |
| [UI 상태 패턴](/design/ui-states/) | 로딩, 에러, 빈 상태, 폼 유효성 등 상태별 UI 처리 기준 |
| [아이콘 시스템](/design/icon-system/) | 8pt 그리드 기반 아이콘 크기, 라이브러리, 접근성 규칙 |
| [프론트엔드 미학](/design/aesthetics/) | AI 슬롭 회피, 5트랙 디자인 사고, 제품별 UI 스타일 매핑 |

## 핵심 원칙

1. **토큰 우선**: 모든 디자인 값은 토큰으로 정의하고 참조한다. 하드코딩 값을 사용하지 않는다.
2. **접근성 기본**: KWCAG/WCAG 2.1 AA 기준을 모든 디자인 의사결정의 전제 조건으로 둔다.
3. **맥락 적합**: 제품 유형(B2B 관리 도구, 공공 시스템, 마케팅 페이지)에 따라 톤과 스타일을 조정한다.
4. **일관성 확보**: 같은 목적의 컴포넌트는 같은 규격을 사용한다. 예외는 문서화한다.
