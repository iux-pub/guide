INFOMIND UX 디자인/퍼블리싱 가이드 시스템에 오신 것을 환영합니다.

이 가이드는 인포마인드 UX팀의 디자인 및 퍼블리싱 규칙을 체계화한 팀 표준입니다. 신규 프로젝트 시작 시 검증된 규칙을 즉시 적용할 수 있도록 구성되어 있습니다.

## 디자인 토큰

색상, 타이포그래피, 간격, 그리드 등 프로젝트 전반에 사용되는 디자인 토큰을 정의합니다. CSS Custom Properties 기반으로 프로젝트별 오버라이드가 가능합니다.

- [토큰 개요](/tokens/) -- 전체 토큰 구조와 사용법

## 컨벤션

BEM 네이밍 규칙, ITCSS 기반 SCSS 구조, 코딩 스타일 등 팀이 합의한 코드 작성 규칙을 안내합니다.

- [컨벤션 개요](/conventions/) -- BEM, SCSS 구조, 코딩 스타일

## 컴포넌트

버튼, 폼, 카드, 테이블, 모달, 탭, 페이지네이션, 브레드크럼 등 공통 UI 컴포넌트의 마크업과 스타일 가이드를 제공합니다. 라이브 미리보기와 코드 예제를 확인할 수 있습니다.

- [컴포넌트 개요](/components/) -- 전체 컴포넌트 목록과 미리보기

## 접근성

KWCAG/WCAG 2.1 AA 기준의 웹 접근성 체크리스트와 컴포넌트별 접근성 가이드를 제공합니다. 공공기관 납품 요건을 충족하는 마크업 패턴을 안내합니다.

- [접근성 개요](/accessibility/) -- 체크리스트와 컴포넌트별 가이드

## 피그마 컨벤션

디자이너를 위한 피그마 작업 규칙입니다. 컴포넌트 네이밍, 레이어 구조, Variable 관리, Auto Layout 패턴을 정의합니다.

- [피그마 컨벤션 개요](/figma/) -- 피그마 작업 규칙 전체 안내

## 디자인 QA

피그마 시안에서 퍼블리싱으로 핸드오프하기 전 품질을 점검하는 가이드입니다.

- [디자인 QA](/design-qa/) -- 핸드오프 전 품질 점검 체크리스트

## 퍼블리싱 심화

시맨틱 마크업, 이미지 최적화, CSS 성능, 애니메이션 등 퍼블리싱 심화 주제를 다룹니다.

- [퍼블리싱 심화 가이드](/guides/) -- 마크업, 이미지, 성능, 애니메이션

## 테스트

크로스 브라우저 테스트, 모바일/터치 테스트, CSS 회귀 테스트 전략을 안내합니다.

- [테스트 개요](/testing/) -- 브라우저, 모바일, CSS 회귀 테스트

## 거버넌스

컴포넌트 버전 정책, 라이프사이클 관리, 변경 승인 프로세스, 기여 가이드를 제공합니다.

- [거버넌스 개요](/governance/) -- 버전 정책, 라이프사이클, 기여 가이드

## 온보딩

신규 팀원을 위한 시작 가이드와 피그마-코드 핸드오프 규칙을 안내합니다.

- [온보딩 개요](/onboarding/) -- 시작 가이드, 핸드오프 규칙
