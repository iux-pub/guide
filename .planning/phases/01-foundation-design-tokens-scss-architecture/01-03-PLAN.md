섹션별 내용:

**1. 색상 팔레트 (Color Palette)**
- Primary 색상 3개 (primary, primary-light, primary-dark) — 각각 정사각형 색상 스와치 + 토큰명 + HEX값
- Gray scale 10단계 (gray-900 ~ gray-50) — 그리드로 배열
- Semantic 4색 (danger, warning, success, info) — 스와치 + 토큰명
- Text 색상 3개 (text, text-secondary, text-disabled)
- Background 2개, Border 2개

각 스와치는 `background-color: var(--color-xxx)` 사용, 아래에 토큰명과 값 표시.

**2. 타이포그래피 스케일 (Typography Scale)**
- font-size 7단계를 실제 크기로 렌더링: 각 단계를 `<p style="--fs: var(--font-size-xxx)">` 없이, CSS 클래스를 사용하여 `font-size: var(--font-size-xxx)` 적용. 인라인 스타일 대신 `<style>` 태그에 `.demo-font-2xl { font-size: var(--font-size-2xl); }` 등 정의.
- font-weight 4단계 미리보기
- line-height 3단계 미리보기 (동일 텍스트 블록에 다른 line-height 적용)
- font-family 표시

**3. 간격 스케일 (Spacing Scale)**
- 7단계(xs~3xl)를 시각적 박스로 표현: 각 단계를 `width`와 `height`가 해당 spacing 값인 색상 박스로 렌더링
- 토큰명, px 값, rem 값 표시

**4. 기타 토큰 (Miscellaneous)**
- Border Radius: 동일 크기 박스에 각 radius 값 적용
- Box Shadow: 동일 카드에 각 shadow 값 적용
- Transition: hover 시 각 속도로 색상 변환 데모
- Z-index: 텍스트 목록으로 스케일 표시 (시각화 불필요)

**5. 그리드 시스템 (Grid System)**
- .container + .grid 데모: 12컬럼 그리드에서 다양한 col-span 조합

모든 색상 스와치에 `aria-label` 추가 (접근성). 텍스트 대비 4.5:1 이상 유지.

문서 구조:

```markdown
# SCSS 파일 구조 가이드

## 개요
ITCSS(Inverted Triangle CSS) 기반의 SCSS 아키텍처. 낮은 specificity에서 높은 specificity 순서로 레이어를 쌓아 CSS 캐스케이드 충돌을 방지한다.

## 폴더 구조
src/scss/ 하위 전체 트리 구조 표시 (파일명 + 한 줄 설명)

## 레이어별 설명

### 1-settings (설정)
- 역할: 토큰(CSS Custom Properties)과 SCSS 변수 정의
- CSS 출력: :root 블록만 출력
- 파일 목록과 각 파일 역할
- 새 토큰 추가 시 이 폴더에 파일 생성

### 2-tools (도구)
- 역할: 믹스인과 함수 — CSS를 직접 출력하지 않음
- CSS 출력: 없음 (사용되는 곳에서 출력)
- 파일 목록과 각 파일 역할
- 새 믹스인 추가 시 이 폴더에

### 3-generic (제네릭)
- 역할: reset, normalize, box-sizing 등 전역 기본값
- CSS 출력: 첫 번째 실제 CSS 출력 레이어
- 수정 빈도: 거의 없음

### 4-elements (엘리먼트)
- 역할: HTML 태그 자체의 기본 스타일 (클래스 없는 셀렉터)
- 주의: .클래스 셀렉터 사용 금지, 태그 셀렉터만

### 5-objects (오브젝트)
- 역할: 구조적 레이아웃 패턴 (.container, .grid)
- 특징: 시각적 스타일(색상, 테두리) 없이 구조만

### 6-components (컴포넌트)
- 역할: BEM 네이밍의 UI 컴포넌트 (.btn, .card, .modal 등)
- Phase 3에서 추가 예정
- BEM 네이밍 필수: .block__element--modifier

### 7-utilities (유틸리티)
- 역할: 단일 목적 헬퍼 클래스 (.sr-only, .hidden)
- 특징: 가장 높은 specificity, 최후의 수단

## 파일 추가 규칙
- 새 파일은 해당 레이어 폴더에 `_filename.scss` 생성
- 해당 레이어의 `_index.scss`에 `@forward 'filename'` 추가
- style.scss는 레이어 단위로 @use하므로 직접 수정 불필요

## 모듈 시스템 (@use/@forward)
- @import 사용 금지 (deprecated)
- 레이어 간 참조: @use '../1-settings/breakpoints' as bp;
- @forward로 모아서 @use로 가져오기 패턴 설명

## 토큰 사용법
- CSS Custom Properties: var(--color-primary), var(--spacing-md) 등
- SCSS 변수: $breakpoint-tablet, $breakpoint-pc (미디어쿼리 전용)
- 하드코딩 금지: color: #1886d8 → color: var(--color-primary)

## 프로젝트별 커스텀
- _project-overrides.scss에서 :root 오버라이드
- 예시 코드 포함

## 빌드 명령어
- npm run build:css — 빌드
- npm run watch:css — 워치 모드
```
