디자인 시안과 퍼블리싱 결과물 사이의 허용 가능한 오차 범위를 정의한다. 이 기준은 디자이너와 퍼블리셔가 합의한 팀 표준이며, 핸드오프 검수 시 기준으로 사용한다.

## 기본 원칙

1. **토큰 사용 시 오차 0**: 디자인과 코드 모두 동일한 토큰을 사용하면 값이 정확히 일치한다. 토큰 기반 작업이 오차를 줄이는 가장 효과적인 방법이다.
2. **일관성 우선**: 개별 요소의 1px 차이보다, 동일한 패턴이 페이지 전체에서 일관되게 적용되는 것이 더 중요하다.
3. **브라우저 렌더링 차이 인정**: 브라우저별 렌더링 엔진 차이로 인한 서브픽셀 차이(0.5px 이내)는 허용한다.
4. **반응형 유연성**: 고정 레이아웃이 아닌 유동적(fluid) 레이아웃에서는 정확한 픽셀 매칭보다 비율과 토큰 준수를 우선한다.

---

## 간격 (Spacing) 허용 오차

패딩, 마진, 갭 등 요소 간 간격에 대한 허용 범위이다.

| 항목 | 허용 오차 | 조건 | 비고 |
|------|:---------:|------|------|
| 토큰 기반 간격 | **0px** | 토큰 사용 시 | 토큰 값이 정확히 일치해야 한다 |
| 컴포넌트 내부 패딩 | **+/- 1px** | 토큰 미사용 시 | 토큰 스케일에 가장 가까운 값으로 맞춘다 |
| 요소 간 마진 | **+/- 2px** | 레이아웃 간격 | 시각적으로 동일해 보이면 허용 |
| Auto Layout 갭 | **0px** | 토큰 매핑 시 | 디자인 토큰과 CSS gap이 1:1 매핑 |
| 그리드 거터 | **0px** | 토큰 기반 | 모바일 16px, 태블릿+ 24px 정확히 일치 |

### 간격 검증 방법

1. 브라우저 개발자 도구(DevTools)에서 computed 값 확인
2. CSS에서 `var(--spacing-*)` 토큰 사용 여부 확인
3. 동일 패턴 반복 시 모든 인스턴스가 같은 간격인지 일괄 확인

## 폰트 (Typography) 허용 오차

폰트 크기, 행간, 자간, 굵기에 대한 허용 범위이다.

| 항목 | 허용 오차 | 조건 | 비고 |
|------|:---------:|------|------|
| font-size | **0** | 토큰 스케일 필수 | 12/14/16/20/24/28/32px 중 하나여야 한다 |
| font-weight | **0** | 토큰 값 필수 | 400/500/600/700 중 하나여야 한다 |
| line-height | **0** | 토큰 값 필수 | 1.2/1.6/1.8 중 하나여야 한다 |
| letter-spacing | **+/- 0.5px** | 디자인 지정 시 | 미지정 시 브라우저 기본값(normal) 사용 |
| font-family | **0** | 동일 폰트 필수 | Pretendard GOV + 폴백 체인 일치 |

### 폰트 검증 방법

1. DevTools에서 computed font-size, font-weight, line-height 확인
2. CSS에서 `var(--font-size-*)`, `var(--font-weight-*)`, `var(--leading-*)` 토큰 사용 여부 확인
3. 특히 heading 계층(h1~h6)이 토큰 스케일에 맞는지 확인

### 주의 사항

- 피그마에서 line-height를 `%` 또는 `px`로 설정한 경우, CSS의 unitless 값(1.6 등)으로 변환 시 소수점 반올림 차이가 발생할 수 있다. 토큰 값(1.2/1.6/1.8)으로 통일한다.
- 62.5% 트릭(1rem = 10px) 환경에서 rem 변환 시 소수점 차이가 없는지 확인한다.

## 컬러 (Color) 허용 오차

색상 값에 대한 허용 범위이다. 색상은 원칙적으로 정확히 일치해야 한다.

| 항목 | 허용 오차 | 조건 | 비고 |
|------|:---------:|------|------|
| 토큰 색상 | **정확히 일치** | 토큰 사용 시 | HEX 값 완전 일치 필수 |
| Primary / Semantic | **정확히 일치** | 브랜드/시맨틱 색상 | 오차 절대 불허 |
| Gray scale | **정확히 일치** | 토큰 기반 | 토큰 외 회색 값 사용 금지 |
| 그라데이션 | **시각적 일치** | 브라우저 차이 허용 | 시작/끝 색상은 정확히 일치, 중간 렌더링은 브라우저 의존 |
| 투명도(opacity) | **+/- 0.05** | rgba/opacity 값 | 0.1 단위로 맞추되 미세 차이 허용 |
| box-shadow 색상 | **시각적 일치** | rgba 값 | 투명도 포함 색상은 시각적 동일성 기준 |

### 컬러 검증 방법

1. DevTools에서 computed color 값 확인 (HEX 또는 RGB)
2. CSS에서 `var(--color-*)` 토큰 사용 여부 확인
3. 하드코딩된 HEX 값이 없는지 SCSS 파일 검색
4. 접근성: 텍스트-배경 대비가 4.5:1 이상인지 도구로 확인

### 절대 불허 항목

다음 항목은 오차를 허용하지 않는다:

- Primary 브랜드 색상 (`--color-primary`: #256ef4)
- Semantic 색상 (`--color-danger`, `--color-warning`, `--color-success`, `--color-info`)
- 텍스트 기본 색상 (`--color-text`: #1e2124)
- 배경 기본 색상 (`--color-bg`: #fff)

## 기타 스타일 허용 오차

border-radius, shadow, transition 등 기타 스타일 속성에 대한 허용 범위이다.

| 항목 | 허용 오차 | 조건 | 비고 |
|------|:---------:|------|------|
| border-radius | **0** | 토큰 값 필수 | 4/8/12/16/9999px 중 하나 |
| border-width | **0** | 정확히 일치 | 보통 1px 사용 |
| border-color | **정확히 일치** | 토큰 사용 | `--color-border` 또는 `--color-border-light` |
| box-shadow | **시각적 일치** | 토큰 사용 | 토큰 정의된 shadow-sm/base/lg 사용 |
| transition 속도 | **+/- 0.05s** | 토큰 기반 | 0.1s/0.3s/0.5s 토큰 우선 |
| z-index | **정확히 일치** | 토큰 값 필수 | 100/200/300/400/500/600 체계 준수 |

## 반응형 허용 범위

반응형 레이아웃에서의 허용 범위이다.

| 항목 | 허용 오차 | 비고 |
|------|:---------:|------|
| 브레이크포인트 | **0** | 768px, 1280px 정확히 일치 |
| 컨테이너 max-width | **0** | PC에서 1200px 정확히 일치 |
| 컬럼 비율 | **+/- 1%** | 유동 그리드에서 퍼센트 반올림 차이 허용 |
| 요소 표시/숨김 | **정확히 일치** | 특정 브레이크포인트에서 요소 표시 여부는 디자인과 일치 |

## 검증 도구

| 도구 | 용도 | 사용 방법 |
|------|------|-----------|
| Chrome DevTools | 모든 CSS 값 확인 | Elements 탭 > Computed 패널에서 실제 적용 값 확인 |
| PerfectPixel 확장 | 오버레이 비교 | 디자인 이미지를 브라우저 위에 오버레이하여 차이 확인 |
| PixelParallel 확장 | 오버레이 비교 | PerfectPixel 대안, 여러 디자인 동시 비교 |
| Contrast Checker | 색상 대비 확인 | WebAIM Contrast Checker 또는 DevTools 내장 대비 도구 |
| CSS Overview (DevTools) | 사용된 색상/폰트 목록 | DevTools > More tools > CSS Overview에서 전체 사용 현황 확인 |

## 오차 발견 시 대응

1. **토큰 미사용이 원인인 경우**: 코드에서 하드코딩 값을 토큰으로 교체한다.
2. **디자인 토큰 누락이 원인인 경우**: 디자이너에게 피드백하여 Variable 연결을 요청한다.
3. **브라우저 렌더링 차이인 경우**: 허용 범위 내이면 수용한다. 범위 초과 시 CSS 조정으로 해결한다.
4. **토큰 스케일에 없는 값인 경우**: 가장 가까운 토큰 값으로 맞추고 디자이너와 협의한다.
