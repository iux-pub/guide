---
title: 핸드오프 QA 체크리스트
order: 2
---

디자이너가 퍼블리셔에게 시안을 전달하기 전에 반드시 점검해야 할 항목을 정리한다. 이 체크리스트를 완료한 후 핸드오프하면 수정 요청 왕복을 최소화할 수 있다.

## 사용법

1. 핸드오프 전 아래 체크리스트를 순서대로 점검한다
2. 각 항목을 확인하며 체크 표시한다 (피그마 파일 내 체크리스트 컴포넌트 또는 별도 시트 활용)
3. 모든 필수 항목을 통과해야 핸드오프 가능하다
4. 권장 항목은 가능한 한 완료하되, 일정상 스킵 시 퍼블리셔에게 사유를 공유한다

---

## 1. 토큰 준수 점검

디자인에 사용된 모든 값이 팀 토큰 체계를 따르는지 확인한다.

### 색상

- [ ] 모든 색상이 피그마 Variable(토큰)을 참조하고 있다
- [ ] 하드코딩된 HEX/RGB 값이 없다 (Variable 미연결 색상 없음)
- [ ] Primary, Gray scale, Semantic 색상이 팀 토큰과 일치한다
- [ ] 텍스트 색상이 토큰을 사용한다 (`Color/Text`, `Color/Text/Secondary`, `Color/Text/Disabled`)

### 타이포그래피

- [ ] 폰트 패밀리가 Pretendard GOV이다 (예외 시 명시적으로 표기)
- [ ] 폰트 크기가 토큰 스케일(12/14/16/20/24/28/32px)에 맞다
- [ ] font-weight가 토큰 값(400/500/600/700) 중 하나이다
- [ ] line-height가 토큰 값(1.2/1.6/1.8) 중 하나이다

### 간격

- [ ] 패딩/마진 값이 토큰 스케일(4/8/16/24/32/48/64px)에 맞다
- [ ] Auto Layout의 padding과 gap이 토큰 값을 사용한다
- [ ] 토큰 스케일에 없는 임의 간격 값이 없다

### 그리드

- [ ] 컨테이너 최대 너비가 1200px이다 (예외 시 명시)
- [ ] 12컬럼 그리드를 기반으로 레이아웃이 구성되어 있다
- [ ] 거터(gutter) 값이 토큰과 일치한다 (모바일 16px, 태블릿+ 24px)

---

## 2. 컴포넌트 일관성 점검

컴포넌트가 라이브러리 규격에 맞게 사용되었는지 확인한다.

### 네이밍

- [ ] 모든 컴포넌트 인스턴스가 라이브러리 원본과 연결되어 있다 (detach 없음)
- [ ] 로컬에서 수정한 컴포넌트가 있다면 사유가 명시되어 있다
- [ ] 컴포넌트 네이밍이 PascalCase 슬래시 계층을 따른다 (예: `Button/Primary/Large`)

### 상태

- [ ] 인터랙티브 요소(버튼, 입력 필드, 링크)의 모든 상태가 디자인되어 있다:
  - Default, Hover, Active/Pressed, Focus, Disabled
- [ ] 폼 필드의 유효성 상태가 디자인되어 있다 (유효, 에러, 경고)
- [ ] 빈 상태(empty state)와 로딩 상태가 필요한 곳에 디자인되어 있다

### 반응형

- [ ] 3개 브레이크포인트(모바일 360px, 태블릿 768px, PC 1280px)에 대한 시안이 있다
- [ ] 브레이크포인트 간 레이아웃 변화가 명확히 표현되어 있다
- [ ] 모바일에서 숨기거나 변경되는 요소가 명시되어 있다

---

## 3. 접근성 점검

디자인 단계에서 확인 가능한 접근성 항목을 점검한다.

### 색상 대비

- [ ] 일반 텍스트(16px 미만)의 배경 대비가 4.5:1 이상이다
- [ ] 큰 텍스트(24px 이상, 또는 bold 19px 이상)의 배경 대비가 3:1 이상이다
- [ ] UI 컴포넌트(버튼, 입력 필드 테두리 등)의 배경 대비가 3:1 이상이다
- [ ] 링크가 주변 텍스트와 색상 외의 방법(밑줄 등)으로도 구분된다

### 포커스 표시

- [ ] 포커스 상태가 모든 인터랙티브 요소에 디자인되어 있다
- [ ] 포커스 표시가 2px 이상의 시각적 변화를 제공한다
- [ ] 포커스 색상이 배경과 3:1 이상의 대비를 가진다

### 터치 타겟

- [ ] 모바일 터치 타겟이 최소 44x44px (WCAG) / 48x48px (권장) 이상이다
- [ ] 인접한 터치 타겟 간 최소 8px 이상의 간격이 있다

### 콘텐츠

- [ ] 이미지에 대체 텍스트(alt) 설명이 디자인 시안에 명시되어 있다
- [ ] 아이콘만 있는 버튼에 레이블 텍스트가 함께 명시되어 있다
- [ ] 색상만으로 정보를 전달하는 요소가 없다 (아이콘, 패턴 등 보조 수단 병행)

---

## 4. 레이어/정리 점검

파일 상태가 핸드오프에 적합한지 확인한다.

### 레이어 정리

- [ ] 숨겨진(hidden) 레이어 중 불필요한 것이 삭제되었다
- [ ] 사용하지 않는 시안/대안(deprecated) 프레임이 별도 페이지로 분리되어 있다
- [ ] 모든 레이어/프레임에 의미 있는 이름이 부여되어 있다 (Frame 1, Group 2 등 기본명 없음)

### 프레임 구조

- [ ] 각 페이지/화면이 독립된 프레임으로 구성되어 있다
- [ ] 프레임이 피그마 컨벤션의 네이밍 규칙을 따른다
- [ ] 핸드오프 대상 프레임이 명확히 구분되어 있다 (예: "Ready" 라벨)

### 설명/주석

- [ ] 특수한 인터랙션이나 동작에 대한 설명이 주석(comment) 또는 텍스트로 표기되어 있다
- [ ] 퍼블리셔가 알아야 할 예외 사항이 명시되어 있다
- [ ] 애니메이션/트랜지션이 필요한 경우 동작 설명이 포함되어 있다

---

## 5. 반응형/적응형 점검

반응형 레이아웃이 올바르게 설계되었는지 확인한다.

### 브레이크포인트

- [ ] 모바일(360px), 태블릿(768px), PC(1280px) 시안이 모두 존재한다
- [ ] 각 브레이크포인트에서 그리드(컬럼 수, 거터, 마진)가 토큰과 일치한다:
  - 모바일: 4컬럼, 거터 16px, 마진 16px
  - 태블릿: 12컬럼, 거터 24px, 마진 24px
  - PC: 12컬럼, 거터 24px, 마진 40px, 최대폭 1200px

### 레이아웃 변화

- [ ] 네비게이션의 반응형 동작이 명시되어 있다 (햄버거 메뉴 등)
- [ ] 사이드바의 반응형 동작이 명시되어 있다 (접힘/하단 이동 등)
- [ ] 카드/리스트의 컬럼 수 변화가 브레이크포인트별로 정의되어 있다
- [ ] 테이블의 반응형 처리 방식이 명시되어 있다 (스크롤/스택 등)
