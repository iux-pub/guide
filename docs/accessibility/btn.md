# 버튼 접근성 가이드

> 스니펫 코드는 `src/snippets/btn.md` 참조. 이 문서는 접근성 심화 가이드(왜/언제/주의사항).

## 필수 ARIA 속성

| 속성 | 대상 요소 | 값 | 설명 |
|------|----------|-----|------|
| `aria-label` | 아이콘 버튼 | 동작 설명 텍스트 | 텍스트 없이 아이콘만 있는 버튼에 필수. 스크린리더가 버튼의 용도를 알 수 있도록 함 |
| `aria-disabled="true"` | 조건부 비활성 버튼 | `true` | 폼 유효성 검증 전 제출 버튼 등에 사용. `disabled`와 달리 포커스는 가능 |
| `aria-expanded` | 토글 버튼 | `true` / `false` | 드롭다운, 아코디언 등 열림/닫힘 상태 버튼에 사용 |
| `aria-pressed` | 토글 상태 버튼 | `true` / `false` | 좋아요, 북마크 등 on/off 상태 버튼에 사용 |

## 키보드 상호작용

| 키 | 동작 |
|----|------|
| `Enter` | 버튼 활성화 (클릭과 동일) |
| `Space` | 버튼 활성화 (클릭과 동일) |
| `Tab` | 다음 포커스 가능 요소로 이동 |
| `Shift+Tab` | 이전 포커스 가능 요소로 이동 |

## `<button>` vs `<a>` 선택 기준

**왜 구분해야 하는가:** 스크린리더는 `<button>`을 "버튼"으로, `<a>`를 "링크"로 읽는다. 역할이 다르면 사용자가 예상하는 동작도 달라진다.

| 상황 | 올바른 태그 | 이유 |
|------|-----------|------|
| 동작 실행 (저장, 삭제, 제출) | `<button>` | 페이지 이동 없이 동작 수행 |
| 페이지 이동 | `<a href="...">` | URL이 변경되는 네비게이션 |
| 모달 열기, 드롭다운 토글 | `<button>` | UI 상태 변경, 페이지 이동 아님 |
| 외부 링크 | `<a href="..." target="_blank">` | 새 창/탭 열기 |

## Do / Don't

### Do (올바른 예)

```html
<!-- 아이콘 버튼에 aria-label 제공 -->
<button type="button" class="btn btn--ghost" aria-label="검색">
  <svg>...</svg>
</button>

<!-- 토글 버튼에 상태 표시 -->
<button type="button" class="btn" aria-expanded="false" aria-controls="dropdown-menu">
  메뉴
</button>
```

**왜:** 스크린리더가 "검색, 버튼"으로 읽어 용도를 즉시 파악할 수 있다. `aria-expanded`는 열림/닫힘 상태를 전달한다.

### Don't (잘못된 예)

```html
<!-- div를 버튼으로 사용 -->
<div class="btn btn--primary" onclick="save()">저장</div>

<!-- a 태그를 버튼처럼 사용 -->
<a href="#" class="btn" onclick="deleteItem()">삭제</a>

<!-- 아이콘 버튼에 대체 텍스트 없음 -->
<button type="button" class="btn btn--ghost">
  <svg>...</svg>
</button>
```

**왜 잘못인가:**
- `<div>`는 키보드로 접근 불가하고 스크린리더가 "버튼"으로 인식하지 못한다
- `<a href="#">`는 스크린리더가 "링크"로 읽어 사용자가 페이지 이동을 기대한다
- 아이콘만 있는 버튼은 스크린리더가 아무 정보도 전달하지 못한다 (빈 버튼)

## `disabled` vs `aria-disabled` 사용 기준

| 속성 | 포커스 가능 | 폼 제출 포함 | 사용 시점 |
|------|-----------|------------|----------|
| `disabled` | 불가 | 제외 | 명확히 비활성인 버튼 (예: 이전 단계 미완료) |
| `aria-disabled="true"` | 가능 | 포함 | 비활성 이유를 설명해야 할 때 (포커스 후 안내 메시지) |

**왜 구분이 필요한가:** `disabled` 버튼은 Tab 키로 도달할 수 없어 스크린리더 사용자가 버튼의 존재 자체를 모를 수 있다. 비활성 이유를 안내해야 하는 경우 `aria-disabled`를 사용하고 클릭 이벤트를 JS에서 막는다.

## 포커스 스타일 주의사항

- `outline: none`을 절대 사용하지 않는다 (키보드 사용자가 현재 위치를 잃는다)
- `:focus-visible` 선택자를 사용하여 키보드 포커스 시에만 아웃라인을 표시한다
- 포커스 아웃라인은 배경과 3:1 이상 대비를 유지해야 한다

## 스크린리더 테스트 노트

- **NVDA:** "저장, 버튼" / 아이콘 버튼: "검색, 버튼" / 비활성: "저장, 버튼, 사용할 수 없음"
- **VoiceOver (macOS):** "저장, 버튼" / 아이콘 버튼: "검색, 버튼" / 비활성: "저장, 흐리게 표시, 버튼"
- **토글 버튼:** "메뉴, 접힘, 버튼" -> 클릭 후 "메뉴, 펼침, 버튼"
