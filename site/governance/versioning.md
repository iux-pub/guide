가이드 시스템은 시맨틱 버전(Semantic Versioning)을 따른다. 변경의 영향 범위에 따라 버전 번호를 부여하고, 모든 변경 사항을 CHANGELOG에 기록한다.

## 시맨틱 버전 형식

```
MAJOR.MINOR.PATCH
```

예: `1.0.0`, `1.2.3`, `2.0.0`

## 변경 수준별 기준

### MAJOR (주요 변경)

기존 프로젝트에 **호환성을 깨뜨리는 변경**이다. 기존 코드 수정이 필요할 수 있다.

| 변경 유형 | 예시 |
|-----------|------|
| 토큰 체계 변경 | 기존 토큰 이름 변경 (`--color-primary` -> `--color-brand`) |
| 토큰 삭제 | `--spacing-xs` 삭제 |
| 컴포넌트 삭제 | deprecated 컴포넌트의 최종 삭제 |
| 브레이크포인트 변경 | 태블릿 기준 768px -> 800px 변경 |
| BEM 네이밍 규칙 변경 | 구분자 규칙 변경 |

### MINOR (기능 추가)

기존 코드에 **영향 없이** 새로운 기능을 추가한다.

| 변경 유형 | 예시 |
|-----------|------|
| 새 컴포넌트 추가 | accordion, tooltip 등 |
| 새 토큰 추가 | `--spacing-4xl` 추가 |
| 새 유틸리티 클래스 추가 | `.text-center`, `.flex-wrap` 등 |
| 새 믹스인 추가 | `@mixin grid-auto-fit` 등 |
| 컴포넌트 변형 추가 | `.btn--outline` 등 |

### PATCH (패치)

기존 동작에 **변화 없는** 수정이다.

| 변경 유형 | 예시 |
|-----------|------|
| 문서 수정 | 오타 수정, 설명 보완 |
| 버그 수정 | 스타일 렌더링 오류, 접근성 속성 누락 수정 |
| 코드 예제 개선 | 스니펫 예제 추가, 코드 정리 |

## CHANGELOG 작성 규칙

[Keep a Changelog](https://keepachangelog.com) 표준 형식을 따른다.

### 파일 위치

프로젝트 루트: `CHANGELOG.md`

### 카테고리

| 카테고리 | 설명 | 예시 |
|----------|------|------|
| **Added** | 새로 추가된 기능 | 새 컴포넌트, 새 토큰, 새 문서 페이지 |
| **Changed** | 기존 기능의 변경 | 토큰 값 수정, 컴포넌트 마크업 변경 |
| **Deprecated** | 향후 제거 예정인 기능 | deprecated 라벨링된 컴포넌트 |
| **Removed** | 삭제된 기능 | MAJOR 업데이트에서 삭제된 토큰/컴포넌트 |
| **Fixed** | 버그 수정 | 접근성 속성 누락 수정, 스타일 오류 수정 |

### 작성 형식

```markdown
## [1.1.0] - 2026-04-15

### Added
- accordion 컴포넌트 추가 (HTML+SCSS 스니펫, 접근성 가이드)

### Changed
- btn 컴포넌트 호버 스타일 개선

### Fixed
- modal 컴포넌트 포커스 트랩 오류 수정
```

### 규칙

1. 날짜 형식: `YYYY-MM-DD`
2. 최신 버전이 맨 위에 위치
3. `[Unreleased]` 섹션에 아직 릴리스되지 않은 변경 사항을 기록
4. 각 항목은 한 줄로 간결하게 작성
5. 영향받는 컴포넌트/토큰 이름을 명시

## 버전 히스토리

전체 변경 이력은 프로젝트 루트의 [CHANGELOG.md](/CHANGELOG.md)를 참조한다.
